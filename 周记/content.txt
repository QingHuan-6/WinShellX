1.分析命令解释程序与内核的关系 
命令解释程序是用户和系统内核之间的接口程序。对于 Windows 系统来说， Command 程序是一个命令语言解释器，它拥有自己内建的命令集，用户或其它应用程序都可通过对 Command 程序的调用完成与系统内核的交互, 硬件、内核、系统调用以及 Command 之间的层次关系如图1所示:

2.学习Windows的系统调用
操作系统把所能够完成的功能以函数的形式供给应用程序使用，应用程序对这些函数的调用叫做系统调用，这些函数的集合就是 Windows 操作系统提供给应用程序编程的接口（Application Programming Interface），简称 Windows API 或 Win32 API。 所有在 Win32 平台上运行的应用程序都可以调用这些函数。 
Microsoft 的所有 32 位平台都支持统一的 API，包括函数、结构、消息、宏及接口。学习Windows 的相关 API，包括定义、使用方法等。 
3.设计一个命令行解释器， 参考 Command 命令解释程序，采用控制台命令行输入，命令提示行是当前目录与提示符（>），在提示符后输入命令，执行结果在控制台中显示。能够执行常用的内部命令，例如 cd、dir、history、exit、tasklist、taskkill 等。 
cd <路径>：切换目录。 
dir：显示指定目录下的文件、目录，及磁盘空间等相关信息。 
history：显示控制台中曾经输入过的命令。 
exit：退出控制台。
tasklist：显示系统当前进程信息，包括进程标识符 pid、线程数、进程名等。 
taskkill ：结束系统中正在运行的进程，需提供进程标识 pid。
